# 安全策略

## 支持范围

当前项目处于早期开放阶段，默认只维护 `main` 分支的最新代码。安全修复会优先合并到 `main`。

## 报告安全问题

请不要在公开 Issue、讨论区或 Pull Request 中披露安全漏洞。报告时请通过仓库维护者公开资料中提供的私密联系方式联系维护者，并尽量包含：

- 漏洞影响范围和风险等级判断。
- 复现步骤或最小复现请求。
- 受影响的分支、提交或版本。
- 相关日志、截图或请求响应示例（请先脱敏）。
- 你认为可行的修复建议。

维护者确认问题后，会尽快评估影响、准备修复并在合适时公开说明。

## 敏感信息处理

- 不要提交 `.env`、真实 API Key、JWT 密钥、数据库密码、生产日志或数据库文件。
- 生产环境必须配置强随机 `JWT_SECRET`。
- 使用用户级 AI API Key 存储能力时，建议配置 `AI_ENCRYPTION_KEY`。
- 开放自定义 AI Provider Base URL 时，建议配置 `AI_BASE_URL_ALLOWLIST`，降低误配置和 SSRF 风险。
- 日志、Issue 和截图中应隐藏邮箱、Token、API Key、账单内容和个人财务数据。

## 已知边界

这个项目适合学习、自托管和二次开发。公网生产部署前，请根据实际环境补充：

- HTTPS 和安全 Cookie 策略。
- 请求限流和登录防爆破。
- 数据备份和恢复流程。
- 日志脱敏、审计和监控告警。
- 数据库账号最小权限。
- 依赖漏洞扫描和镜像安全扫描。
