# 产品需求规格说明书 (PRD)

本文档定义了产品的功能需求与验收标准。请配合 `DEVELOPMENT_ROADMAP.md` 使用：Roadmap 指导“怎么做”，本 PRD 定义“做成什么样”。

---

## 📅 第一阶段：视觉重构 (JARVIS Interface)

### 需求 1.1：票据式结果卡片
*   **用户故事** (User Story):
    > 作为用户，在 AI 识别出一笔交易后，我希望看到一张类似超市小票的卡片，而不是普通的列表，这样我能更直观地核对金额和分类。
*   **功能详情**：
    *   **UI 样式**：
        *   卡片背景为纯白 (`#FFFFFF`)。
        *   边缘效果：左右两侧应有半圆形缺口（模拟撕孔位置），或底部边缘呈锯齿状。
        *   阴影：带有轻微的 `elevation: 2` 或 `shadowOpacity: 0.1`。
    *   **信息层级**：
        *   **第一级 (最醒目)**：金额。支出显示红色 (`#EF4444`)，收入显示绿色 (`#10B981`)。字号需大于 24sp。
        *   **第二级**：分类图标与名称。
        *   **第三级**：日期与备注（灰色小字）。
    *   **交互逻辑**：
        *   点击“编辑”图标：卡片上的文字变为输入框，背景轻微变灰。
        *   点击“删除”图标：卡片向左滑动消失或淡出。

### 需求 1.2：JARVIS 仪表盘 (HUD)
*   **用户故事** (User Story):
    > 作为用户，打开 App 时，我希望直接看到我的财务健康状况（预算条），而不是一堆流水账，时刻提醒自己是否超支。
*   **功能详情**：
    *   **布局重构**：
        *   移除首页原本的 `FlatList` 交易列表。
        *   屏幕上方 60% 区域为状态显示区 (HUD)。
    *   **HUD 元素**：
        *   **预算进度条**：
            *   样式：横向粗条。
            *   颜色逻辑：<50% 绿色 / 50-80% 黄色 / >80% 红色。
            *   动画：数值变化时，进度条应有“回弹”或“震动”效果。
        *   **今日支出**：超大号数字居中显示。
    *   **输入反馈**：
        *   当底部输入框提交成功时，不弹出 Toast，而是 HUD 区域的数据直接跳动更新。

## 🥚 第二阶段：情感化宠物空间 (Pet Space)

### 需求 2.1：独立宠物房间
*   **用户故事** (User Story):
    > 作为用户，我希望有一个单独的、温馨的页面来查看我的 AI 伴侣，在这个页面里我可以放松下来和它交流。
*   **功能详情**：
    *   **入口**：首页下拉，或底部导航栏新增 "Space/空间" Tab。
    *   **场景渲染**：
        *   背景：温馨的室内场景（如客厅或书房）。
        *   角色：屏幕中央显示 AI 形象（可以是静态图或 Lottie 动画）。
    *   **状态映射**：
        *   如果本月预算充足 -> 宠物表情开心/正常。
        *   如果本月预算超支 -> 宠物表情生病/焦虑。

### 需求 2.2：实时语音通话
*   **用户故事** (User Story):
    > 作为用户，在宠物空间里，我想按住按钮就能直接说话，松开就能听到 AI 的语音回复，就像发微信语音一样自然。
*   **功能详情**：
    *   **通话按钮**：
        *   底部居中悬浮一个大按钮（电话图标）。
        *   交互：**按住说话 (Push-to-Talk)**。
    *   **反馈状态**：
        *   按住时：按钮变大，周围有波纹扩散动画。
        *   松开处理中：宠物上方显示“思考气泡 (...)”。
        *   播放回复时：宠物上方显示“声波图”，并自动播放 TTS 音频。

## ⚙️ 第三阶段：智能配置系统 (Smart Config)

### 需求 3.1：自适应模型检测
*   **用户故事** (User Story):
    > 作为用户，我想配置自己的 API Key，并测试它能不能用。如果我买了高级模型（如 GPT-4o），系统应该自动开启高级语音功能。
*   **功能详情**：
    *   **设置项**：增加 "AI Model Config" 区域。
    *   **测试流程**：
        *   用户点击 "测试连接/Test Connection"。
        *   系统尝试建立 Realtime WebSocket 连接。
    *   **判定逻辑**：
        *   **Case A (原生支持)**：提示“✅ 原生语音已激活”，并隐藏 ASR/TTS 选项。
        *   **Case B (仅文本)**：提示“⚠️ 需要配置语音组件”，并自动展开 ASR (Whisper) 和 TTS 设置框。
    *   **数据持久化**：配置通过 `AsyncStorage` 保存，重启 App 不丢失。

## ✅ 验收清单 (Checklist)

在提交代码前，请确保满足以下核心指标：

- [ ] **视觉**：首页没有列表，只有仪表盘。结果卡片是票据样式。
- [ ] **交互**：按住通话按钮说话，松开后在 3 秒内听到语音回复（标准模式下）。
- [ ] **逻辑**：设置页面的“测试”按钮能准确区分模型类型并保存配置。
- [ ] **稳定性**：在未配置 Key 的情况下，进入页面不闪退，而是提示配置。
