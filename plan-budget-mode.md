# 预算系统重构计划：实现独立预算模式

## 1. 背景与目标

当前预算系统允许用户同时设置“总预算”和“分类预算”，导致其相互关系和计算逻辑模糊，给用户带来困惑。

**核心目标**：将预算系统重构为两种**独立且互斥**的模式，让用户选择一种符合自己习惯的、清晰易懂的预算方法。

## 2. 核心设计：独立预算模式

系统将提供两种预算模式供用户选择：

-   **A. 总预算模式 (Total Budget Mode)**
    -   **理念**：用户只关心总支出上限，不关心具体分类。
    -   **行为**：
        -   系统中只允许存在**一个**预算项，即“总预算”。
        -   所有类型的支出都从这个唯一的预算中扣除。
        -   UI 上，“新增预算”功能会变为“设置/修改总预算”。

-   **B. 分类预算模式 (Category Budget Mode)**
    -   **理念**：用户希望对不同消费类别进行精细化管理。
    -   **行为**：
        -   用户可以设置**多个**针对不同分类的预算（如餐饮、交通等）。
        -   系统中**不允许**单独设置一个全局的“总预算”项。
        -   每笔支出根据其分类，从对应的分类预算中扣除。
        -   在“预算概览”中展示的“总预算”，是所有分类预算金额的动态总和。

## 3. 实现策略

### 3.1 模式状态存储

-   **方案**: 使用浏览器的 `localStorage` 来持久化存储用户选择的模式。
-   **键名**: `app_budget_mode`
-   **值**: `'total'` 或 `'category'`。
-   **默认值**: `'category'` (分类预算模式，功能更丰富)。
-   **优点**: 实现简单，无需修改后端数据库和 API，用户选择得以在本地保留。

### 3.2 UI/UX 交互设计

-   **模式切换器**:
    -   **位置**: 在“预算管理”标签页的 `Card` 组件标题区域（`extra` 属性）。
    -   **控件**: 使用 `Radio.Group` 并设置为 `buttonStyle="solid"`，使其看起来像一组切换按钮。
-   **切换确认**:
    -   当用户尝试切换模式时，必须弹出一个 `Modal.confirm` 对话框。
    -   **提示信息**: “切换预算模式将清空所有当前的预算设置，以便开始新的预算方案。您确定要继续吗？”
    -   **确认操作**: 如果用户确认，则调用后端 API **删除所有**当前的预算记录，然后将新模式存入 `localStorage` 并刷新页面。

### 3.3 组件重构范围

1.  **`frontend/src/BudgetManager.tsx` (核心)**
    -   引入 `useState` 来管理当前预算模式 `budgetMode`，并使用 `useEffect` 从 `localStorage` 初始化。
    -   实现模式切换器 `Radio.Group` 及其 `onChange` 事件处理器（包含弹窗确认和清空数据的逻辑）。
    -   根据 `budgetMode` 的值，**条件渲染**整个预算列表的展示逻辑。
        -   在“总预算模式”下，如果已有总预算，则“新增预算”按钮变为“修改总预算”；如果没有，则为“设置总预算”。列表中最多只显示一项。
        -   在“分类预算模式”下，行为与现在类似，但需要过滤掉 `budgetType === 'total'` 的数据。

2.  **`frontend/src/hooks/useBudget.ts`**
    -   修改 `budgetOverview` 的计算逻辑。
        -   传入 `budgetMode` 作为参数。
        -   如果 `budgetMode === 'total'`，`totalBudget` 就是唯一的那个总预算的限额。
        -   如果 `budgetMode === 'category'`，`totalBudget` 是所有分类预算限额的总和。

3.  **`frontend/src/hooks/useBudgetForm.ts`**
    -   在 `openCreateModal` 时，根据当前的 `budgetMode` 自动填充 `budgetType` 字段。
        -   `budgetMode === 'total'` -> `budgetType` 设为 `'total'` 且**只读**。
        -   `budgetMode === 'category'` -> `budgetType` 设为 `'category'` 且**只读**。
    -   这将大大简化新增预算时的用户操作。

## 4. 实施步骤

1.  **规划与设计** (当前步骤)。
2.  **修改数据钩子 (`useBudget.ts`)**: 调整 `budgetOverview` 的计算逻辑。
3.  **重构主组件 (`BudgetManager.tsx`)**: 实现模式状态管理、UI切换器和条件渲染。
4.  **调整表单钩子 (`useBudgetForm.ts`)**: 根据模式预设并锁定 `budgetType`。
5.  **全面验证**: 测试两种模式下的所有功能，特别是模式切换和数据清空逻辑。
