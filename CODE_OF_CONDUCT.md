# 行为准则

本项目希望保持开放、尊重和务实的协作环境。参与讨论、提交 Issue、创建 Pull Request 或维护项目时，请遵守以下原则。

## 我们鼓励的行为

- 尊重不同经验水平、背景和观点。
- 围绕事实、代码和用户影响讨论问题。
- 清楚说明问题、复现步骤、权衡和限制。
- 接受建设性的技术反馈，也给出具体、可执行的反馈。
- 在发现错误时及时修正。

## 不接受的行为

- 人身攻击、骚扰、歧视或威胁。
- 发布他人的隐私信息、密钥、财务数据或未授权资料。
- 使用侮辱性、贬低性或持续干扰讨论的表达。
- 在 Issue 或 Pull Request 中刷屏、引战或偏离主题。
- 明知存在安全风险仍推动不负责任的公开披露。

## 执行方式

维护者可以根据情况编辑、隐藏或删除违反行为准则的内容，也可以限制相关账号参与项目讨论。严重或持续违规会被禁止参与项目协作。

如果你需要报告行为准则问题，请通过仓库维护者公开资料中提供的联系方式联系维护者，并尽量提供相关链接、截图或上下文。
